### 一、Karpathy LLM Wiki 核心原理与架构

Andrej Karpathy 的个人知识库方案核心是**Obsidian+AI Agent 协同工作流**，实现 “人类收集、AI 整理” 的知识复利闭环：

- **Obsidian**：本地存储 + 可视化前端，提供双链、图谱、检索能力（免费版足够）
- **AI Agent**（Claude Code/openCode）：文件读写 + 智能处理，负责**摘要、分类、概念提取、双链建立、索引维护**
- **三层目录结构**（严格区分权限）：
    
    - **`raw/`**：原始资料（只读），存放文章、论文、笔记等，人类只增不改
    - **`wiki/`**：AI 编译后的结构化知识库（AI 全权维护），人类只查不写
    - **`output/`**：查询 / 分析产物（可回流到`wiki/`）

#### 3. 核心 Schema 配置（`CLAUDE.md`，AI 的工作手册）

markdown

```
# LLM Wiki 系统规则（Karpathy 规范）
## 1. 目录权限与职责
- **raw/**：只读区，AI禁止修改任何文件，仅用于读取原始资料
- **wiki/**：AI维护区，AI全权负责创建/更新/删除，人类仅作查阅
- **output/**：临时输出区，优质内容可回流到wiki
- **CLAUDE.md**：系统规则，AI必须严格遵守，可由人类更新

## 2. 页面格式规范（所有.md文件必须包含）
---
title: 页面标题（简洁准确）
type: concept | entity | source | comparison | topic
sources: [raw/xxx.md, raw/yyy.pdf]  # 原始来源引用
related: [[相关页面1]], [[相关页面2]]  # 自动建立的双链
created: YYYY-MM-DD  # 创建时间
updated: YYYY-MM-DD  # 最后更新时间
---
**Summary**: 一句话核心摘要（≤100字）
**Tags**: #主题1 #主题2 #主题3  # 3-5个核心标签

## 3. 三大核心指令（人类发起，AI执行）
### /ingest <路径> - 处理新资料
1. 读取原始文件，提炼核心观点与结论
2. 在 wiki/sources/ 创建源文件摘要页
3. 提取所有概念/实体，在对应目录创建/更新页面
4. 建立跨页面双链关联（相关概念/实体/来源）
5. 更新 wiki/index.md（新增条目）和 wiki/log.md（操作记录）
6. 处理冲突：发现矛盾标注 ⚠️，记录不同观点

### /query <问题> - 知识检索与整合
1. 仅从 wiki/ 读取内容，不访问 raw/
2. 整合相关页面信息，生成结构化回答
3. 所有结论必须标注引用 [[页面名]]
4. 优质结果自动保存到 wiki/topics/ 新页面
5. 更新索引与日志

### /lint - 知识库健康检查
1. 检测内容矛盾 ⚠️，标注位置与原因
2. 查找断链（无入链/出链）与孤立页面
3. 识别过期内容 ⏳（超过6个月未更新）
4. 补全缺失的YAML前置、Summary、Tags
5. 生成检查报告保存到 wiki/lint-report-YYYY-MM-DD.md
6. 更新索引与日志

## 4. 铁律（AI必须严格遵守）
- 每次操作必须更新 wiki/index.md 和 wiki/log.md
- 概念/实体必须独立成页，避免内容冗余
- 所有链接使用 Obsidian 双链格式 [[页面名]]
- 自动执行指令，无需询问确认
- 保持语言一致性（中文/英文），根据原始资料语言选择
```

### 四、日常工作流（5 步循环）

1. **收集资料**：用 Web Clipper 剪藏网页，或手动放入 `raw/` 对应子目录
2. **AI 编译**：发送 `/ingest raw/xxx.md`，AI 自动生成结构化 Wiki 页面
3. **知识查询**：发送 `/query 你的问题`，AI 从 Wiki 整合答案并保存到 `wiki/topics/`
4. **定期维护**：每周发送 `/lint`，AI 自动检查并修复知识库问题
5. **内容沉淀**：将优质 `output/` 内容通过 `/fileback` 回流到 `wiki/`，形成知识复利

---

### 五、关键优化技巧

1. **版本控制**：定期提交 Git，可回滚 AI 错误修改，追踪知识演进历史
2. **提示词迭代**：根据领域特点优化 `CLAUDE.md`，让 AI 更懂你的专业术语
3. **批量处理**：使用 `/ingest raw/articles/` 处理整个目录的新资料
4. **图谱可视化**：在 Obsidian 中打开图谱视图，直观查看知识关联网络
5. **性能优化**：对大文件先手动分割，或让 AI 先生成摘要再处理细节
